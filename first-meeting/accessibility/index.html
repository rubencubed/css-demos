New(ish) HTML/CSS Features that Improve Accessibility

Over the past few months and years, there have been tremendous improvements to CSS and HTML that reduce or eliminate the need to use JavaScript to achieve certain design patterns. These new CSS and HTML options are more semantic and natively manange focus state in the way the web component is intended to be used. Moreover—since this can be a concern for the design oriented folks—there is no need to sacrifice the visual component of the templates. Often, these new patterns actually make it easier to build designs that excite and engage.

Popups with <dialog>

A <dialog> (modal) is hidden by default. When a user activates a button with command="show-modal" and commandfor="[dialog]-id", the modal is opened. The modal can then be closed with the ESC key or by clicking away from the modal (support for an explicit "close" button is not fully cross browser). Focus jumps inside of the modal on opening and reverts to the opening button when the modal is dismissed.
